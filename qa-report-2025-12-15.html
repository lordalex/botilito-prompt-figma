Bug #1: Invalid Session Handling (FIXED)

App stuck on "Cargando..." with invalid refresh token
FIXED Critical

Description

When a user has an expired or invalid refresh token stored in localStorage, the application would get stuck on the loading screen ("Cargando...") indefinitely instead of redirecting to the login page.

Root Cause

The AuthProvider.tsx component's getInitialSession() function lacked proper error handling. When supabase.auth.getSession() failed with an AuthApiError: Invalid Refresh Token, the error was thrown but setIsLoading(false) was never called, leaving the app in a perpetual loading state.

Console Error

AuthApiError: Invalid Refresh Token: Refresh Token Not Found
    at handleError2 (supabase-js.js:5075:9)
    at async _handleRequest2 (supabase-js.js:5116:5)
    at async _request (supabase-js.js:5100:16)

Files Modified

  • src/providers/AuthProvider.tsx (lines 67-93)

Fix Applied

src/providers/AuthProvider.tsx
- const getInitialSession = async () => {
-     const { data: { session } } = await supabase.auth.getSession();
-     setSession(session);
-     setUser(session?.user ?? null);
-     if (session?.user) {
-         await checkUserProfile();
-     }
-     setIsLoading(false);
- };

+ const getInitialSession = async () => {
+     try {
+         const { data: { session }, error } = await supabase.auth.getSession();
+         if (error) {
+             console.error('Session error:', error.message);
+             setSession(null);
+             setUser(null);
+             setIsLoading(false);
+             return;
+         }
+         setSession(session);
+         setUser(session?.user ?? null);
+         if (session?.user) {
+             await checkUserProfile();
+         }
+     } catch (error) {
+         console.error('Failed to get session:', error);
+         setSession(null);
+         setUser(null);
+     } finally {
+         setIsLoading(false);
+     }
+ };

Bug #2: Invalid Date Display (FIXED)

"Fecha de Análisis" showing "Invalid Date"
FIXED Medium

Description

After completing an AI analysis, the "Fecha de Análisis" field displayed "Invalid Date" instead of a properly formatted date.

Root Cause

The API response's created_at field was either undefined or contained an invalid date string. The code directly passed this value to new Date() without validation, resulting in an Invalid Date object when the value was missing or malformed.

Files Modified

  • src/components/ContentAnalysisView.tsx (lines 128-131, 178)
  • src/components/AnalysisResultDisplay.tsx (lines 43-46, 111)

Fix Applied

src/components/ContentAnalysisView.tsx
+ // Safely parse the date with fallback to current date
+ const analysisDate = created_at && !isNaN(new Date(created_at).getTime())
+     ? new Date(created_at)
+     : new Date();

// In the JSX:
- <div>{new Date(created_at).toLocaleString('es-CO')}</div>
+ <div>{analysisDate.toLocaleString('es-CO')}</div>

Verification

After applying the fix, dates now display correctly: "15/12/2025, 8:47:19 a. m."

Note on AI Analysis Accuracy
Observation

Context

During testing with the CNN article about Kristi Noem's visit to Colombia, the AI flagged the content as "Engañoso" (Misleading), claiming that Kristi Noem's title as "Secretaria de Seguridad Nacional" (Secretary of Homeland Security) was incorrect.

Finding

However, Kristi Noem IS the Secretary of Homeland Security, having been appointed in January 2025 under the Trump administration. The AI's assessment appears to be based on outdated knowledge from before her appointment.

Implication

This is not a frontend bug but rather an AI knowledge cutoff issue. The underlying AI model may have a training data cutoff date that predates Kristi Noem's appointment as Secretary of Homeland Security, causing it to incorrectly flag accurate information as misleading.

Recommendation

  • Consider implementing real-time fact-checking against current news sources
  • Add disclaimers about AI knowledge cutoff dates in the analysis results
  • Evaluate updating the AI model or implementing retrieval-augmented generation (RAG) for current events
